# RL-агент для расстановки ворот дрон-рейсинга (v3 — All-at-Once)

## Постановка задачи

Нейросеть, которая **сама учится** расставлять ворота для дрон-рейсинга на арене 10×10м, соблюдая правила. Подход: **Reinforcement Learning** (обучение с подкреплением).

## Ограничения (правила от преподавателя)

| Параметр | Значение |
|---|---|
| Размер арены | 10 × 10 м |
| Мин. расстояние до границы | 3 м |
| Мин. расстояние между соседними воротами | 3 м |
| Макс. расстояние между соседними воротами | 10 м |
| Размер ворот | 1 м |
| Макс. разница углов соседних ворот | 180° |
| Замкнутость | маршрут должен быть кольцевым |

Рабочая зона для центров ворот: [3.0, 7.0] × [3.0, 7.0] м.

### Дополнительные правила (выведены из анализа результатов)

| Правило | Зачем |
|---|---|
| Глобальная мин. дистанция 2 м | Не давать воротам кучковаться |
| Нет пересечений сегментов пути | Маршрут не должен быть «крест-накрест» |
| Нет резких разворотов (> 150°) | Маршрут должен быть плавным |

## Эволюция подходов

### v1 — Supervised Learning (отвергнут)

Генератор создаёт правильные конфигурации → нейросеть копирует. **Проблема:** если генератор уже умеет, зачем нейросеть?

### v2 — RL, последовательная расстановка (отвергнут)

Агент ставит ворота **по одному** (GRU Actor-Critic + PPO). **Результат:** max 14% validity на 3-4 воротах, при добавлении 5+ ворот — обвал. Агент не может планировать наперёд и пересекает собственные пути.

### v3 — RL, всё за один шаг (текущий)

Агент выдаёт **ВСЕ ворота сразу**. Среда оценивает конфигурацию целиком за один шаг.

**Почему лучше:** модели не нужно планировать последовательность — она видит всю задачу целиком, как «рисунок», а не «пошаговую инструкцию».

## Архитектура v3

### Общая схема

```
┌──────────────────┐    action: ВСЕ ворота     ┌──────────────────┐
│                   │    (x1,y1,a1,...,xN,yN,aN) │                  │
│  АГЕНТ (MLP)     │ ──────────────────────────► │  СРЕДА (арена)   │
│  нейросеть       │                             │  с правилами     │
│                   │ ◄───────────────────────── │                  │
│                   │    reward (одно число)      │                  │
└──────────────────┘                             └──────────────────┘
         ▲                                               │
         │              PPO обновляет веса               │
         └───────────────────────────────────────────────┘
```

**Ключевое отличие от v2:** один эпизод = один шаг.

### Нейросеть

```
n_gates (1) → Linear(256) → ReLU
            → Linear(256) → ReLU
            → Linear(256) → ReLU
            ├─ Actor:  Linear(256) → ReLU → Linear(18) → Sigmoid → позиции ворот
            └─ Critic: Linear(128) → ReLU → Linear(1) → оценка состояния
```

- **Actor**: выдаёт 18 чисел = 6 ворот × 3 (x, y, угол) в [0, 1]
- **Critic**: оценивает, насколько хорош текущий запрос (число ворот)
- **Нет GRU** — нет последовательности, нет проблемы «забывания»

### Система наград

| Правило | Бонус | Штраф |
|---|---|---|
| Границы рабочей зоны | +0.3/ворота | −1.0/ворота |
| Дистанция соседних (3–10 м) | +1.5/пара | −2.0/пара |
| ВСЕ дистанции в норме | +5.0 | — |
| Углы ворот (≤ 180°) | +0.5/пара | −1.0/пара |
| Глобальная мин. дистанция (2 м) | — | −2.0/пара |
| Нет пересечений | +3.0 | −2.0/пересечение |
| Нет разворотов (≤ 150°) | +1.0 | −1.0/разворот |
| Coverage (разброс по арене) | +std | — |

## Метрики

- **Reward** — средняя награда за эпизод
- **Validity rate** — % конфигураций, соблюдающих ВСЕ правила
- **Validity by N** — разбивка по числу ворот (3, 4, 5, 6)
