# RL-агент для расстановки ворот дрон-рейсинга

## Постановка задачи

Нужна нейросеть, которая **сама учится** расставлять ворота для дрон-рейсинга на арене 10×10м, соблюдая правила.

**Ключевое отличие от supervised learning:** мы НЕ показываем модели примеры правильных расстановок. Вместо этого модель сама пробует ставить ворота, а мы говорим ей "хорошо" или "плохо" через систему наград (reward).

## Ограничения (правила от преподавателя)

| Параметр | Значение |
|---|---|
| Размер арены | 10 × 10 м |
| Мин. расстояние до границы | 3 м |
| Мин. расстояние между соседними воротами | 3 м |
| Макс. расстояние между соседними воротами | 10 м |
| Размер ворот | 1 м |
| Макс. разница углов соседних ворот | 180° |
| Замкнутость | маршрут должен быть кольцевым |

Рабочая зона для центров ворот: [3.0, 7.0] × [3.0, 7.0] м.

## Почему RL, а не supervised learning?

Supervised learning требует готовый датасет правильных примеров. Но если мы можем генерировать такие примеры алгоритмом — зачем нейросеть? Она просто копирует генератор.

RL решает эту проблему: модель учится **на правилах через reward**, а не на примерах. Она сама экспериментирует, получает обратную связь и открывает стратегии размещения.

**Аналогия:** как ребёнок учится кататься на велосипеде — не по видео (supervised), а пробуя сам: упал → "плохо", проехал → "хорошо".

## Архитектура

### Общая схема (цикл RL)

```
┌─────────────────┐     action: (x, y, angle)     ┌──────────────────┐
│                  │ ─────────────────────────────► │                  │
│   АГЕНТ (GRU)    │                                │  СРЕДА (арена)   │
│   нейросеть      │ ◄───────────────────────────── │  с правилами     │
│                  │     state + reward              │                  │
└─────────────────┘                                └──────────────────┘
```

1. **Среда** — арена с правилами. Принимает действие, проверяет правила, выдаёт reward.
2. **Агент** — GRU-нейросеть. Смотрит на текущее состояние, решает куда ставить ворота.
3. **PPO** — алгоритм обновления весов: "делай больше хорошего, меньше плохого".

### Среда (Environment)

**Состояние (state)** — вектор 7 чисел:
- Прогресс (шаг/всего)
- Последние ворота (x, y, angle) — нормализованные
- Первые ворота (x, y, angle) — для ориентира замыкания

**Действие (action)** — 3 числа [0, 1] → переводятся в реальные координаты

**Система наград:**
| Событие | Reward |
|---|---|
| Расстояние в норме (3–10м) | +1.0 |
| Слишком близко / далеко | −2.0 |
| Угол в норме | +0.5 |
| Угол нарушен | −1.0 |
| Маршрут замкнут | +5.0 |
| Замыкание не удалось | −3.0 |

### Нейросеть агента (Actor-Critic)

```
State (7) → Linear(64) → ReLU → GRU(64) → ┬─ Actor: mean, std → action
                                            └─ Critic: value
```

- **Actor** (актёр) — решает КУДА ставить ворота. Выдаёт параметры нормального распределения, из которого сэмплируется действие.
- **Critic** (критик) — оценивает НАСКОЛЬКО ХОРОШО текущее состояние. Помогает вычислять advantage.
- **GRU** — рекуррентный слой, запоминает контекст предыдущих шагов.

### PPO (Proximal Policy Optimization)

Алгоритм обновления весов:
1. Агент играет 32 эпизода, собирает опыт
2. Вычисляет **advantage** = "это действие было лучше/хуже среднего?"
3. Обновляет веса: P(хорошие действия) ↑, P(плохие действия) ↓
4. **Clip** — ограничивает размер обновления для стабильности

## Метрики

- **Reward** — средняя награда за эпизод (должна расти)
- **Validity rate** — % конфигураций, соблюдающих ВСЕ правила (главная метрика)
