# Авторегрессивная модель расстановки ворот для дрон-рейсинга

## Постановка задачи

**Вход:** количество ворот N (например, 5–10)
**Выход:** последовательность ворот `[(x₁, y₁, α₁), (x₂, y₂, α₂), ..., (xₙ, yₙ, αₙ)]`

Каждые ворота описываются тремя параметрами:
- `x, y` — координаты центра (в метрах)
- `α` — угол наклона (в радианах, [0, 2π))

## Ограничения

| Параметр | Значение |
|---|---|
| Размер арены | 10 × 10 м |
| Размер ворот | 1 м |
| Мин. расстояние до границы | 3 м |
| Мин. расстояние между соседними воротами | 3 м |
| Макс. расстояние между соседними воротами | 10 м |
| Макс. разница углов соседних ворот | 180° |
| Замкнутость | последние ворота → первые тоже 3–10 м |

Рабочая зона: x, y ∈ [3.0, 7.0] м (с учётом отступа от границ).

## Архитектура

### Идея

Авторегрессивная модель (GRU) генерирует ворота **последовательно** — на каждом шаге предсказывает `(x, y, α)` следующих ворот на основе всех предыдущих.

### Схема модели

```
                 ┌─────────────┐
  noise z ──────►│   FC embed   │
                 └──────┬──────┘
                        ▼
  gate_1 ──► GRU ──► GRU ──► ... ──► GRU ──► GRU
              │        │               │        │
              ▼        ▼               ▼        ▼
           gate_2   gate_3          gate_N   (stop)
```

1. **Вход на шаге t:** предыдущие ворота `(x, y, α)` + скрытое состояние GRU
2. **Выход на шаге t:** параметры следующих ворот `(x, y, α)`
3. **Начальное состояние:** случайный вектор `z` (для разнообразия генерации)

### Детали

- **Encoder входа:** Linear(3 → hidden_dim) — проецирует `(x, y, α)` в пространство эмбеддингов
- **GRU:** 2 слоя, hidden_dim = 128
- **Decoder выхода:** Linear(hidden_dim → 3) → активации:
  - `x, y`: sigmoid × 4.0 + 3.0 → диапазон [3.0, 7.0]
  - `α`: sigmoid × 2π → диапазон [0, 2π)

## Этапы работы

### Этап 1. Генератор данных (rule-based)

Генерируем N_samples (10 000+) валидных конфигураций ворот:

```
Алгоритм генерации одной конфигурации:
1. Выбрать случайное N ∈ [5, 10] (кол-во ворот)
2. Поставить первые ворота: случайные (x₁, y₁) ∈ [3, 7], α₁ ∈ [0, 2π)
3. Для i = 2..N:
   a. Сэмплировать расстояние d ∈ [3, 10] и направление θ
   b. Вычислить (xᵢ, yᵢ) = (xᵢ₋₁ + d·cos(θ), yᵢ₋₁ + d·sin(θ))
   c. Сэмплировать αᵢ так, чтобы |αᵢ - αᵢ₋₁| ≤ π
   d. Проверить: в рабочей зоне? не слишком близко к другим?
   e. Если нет — пересэмплировать (до K попыток)
4. Проверить замкнутость: 3 ≤ dist(last, first) ≤ 10
5. Если всё ок — сохранить, иначе — повторить с шага 1
```

**Выход:** массив конфигураций, каждая — список `(x, y, α)` переменной длины.

### Этап 2. Датасет и DataLoader

- Нормализация координат: `(x - 3) / 4` → [0, 1], углов: `α / 2π` → [0, 1]
- Паддинг последовательностей до макс. длины (10)
- Маска для реальных vs padding элементов
- Добавляем первые ворота в конец последовательности (для замкнутости)

### Этап 3. Обучение модели

- **Loss:** MSE между предсказанными и реальными `(x, y, α)` (teacher forcing)
- **Optimizer:** AdamW, lr = 1e-3
- **Scheduler:** CosineAnnealingLR
- **Эпох:** 100, early stopping patience = 15
- **Batch size:** 64
- **Train/Val split:** 80/20

Teacher forcing: на каждом шаге подаём **реальные** предыдущие ворота, модель предсказывает следующие.

### Этап 4. Инференс (генерация новых трасс)

```
Алгоритм генерации:
1. Сэмплировать z ~ N(0, 1)
2. Сэмплировать первые ворота (x₁, y₁, α₁) случайно в рабочей зоне
3. Для i = 2..N:
   a. Подать (xᵢ₋₁, yᵢ₋₁, αᵢ₋₁) в GRU
   b. Получить (x̂ᵢ, ŷᵢ, α̂ᵢ)
   c. Проверить ограничения, при нарушении — clamp/resample
4. Проверить замкнутость (dist к первым воротам)
5. При необходимости — скорректировать последние ворота
```

### Этап 5. Валидация

Метрики качества генерации:
- **Validity rate** — % сгенерированных конфигураций, соблюдающих все ограничения
- **Diversity** — среднее попарное расстояние между конфигурациями
- **Coverage** — насколько равномерно ворота покрывают рабочую зону

### Этап 6. Визуализация

- Отрисовка арены 10×10 с рабочей зоной
- Ворота как отрезки длиной 1м с указанием угла
- Линии между соседними воротами (для наглядности маршрута)
- Подписи расстояний и углов
